Markdown

可能很多小伙伴并不知道Markdown是什么，在这里简述一下Markdown的优缺点

  • 各大平台对于Markdown的解释

    • 维基百科：https://zh.wikipedia.org/wiki/Markdown

    • 百度百科：https://baike.baidu.com/item/markdown/3245829

    • Markdown中文网：https://markdown.cn/

    • Markdown在线编辑器：https://tableconvert.com/

  • 我对于Markdown的感受

    ​ 方便容易的标记语言，可以很轻松的将无序的文字、平庸的排版、无法确认那些是重点标记的文字，通通搞定，而且这种方式可以在文字编辑过程中顺手带入的，而不是需要调整工具栏去选择那些加粗、标题、或者加颜色。

    ​ 最常的协作流程一般都是在 iPhone备忘录 打草稿 → Typora 以Marodown格式写出整篇文章 →分发各自平台。

    ​ 对于支持Markdown的平台，可以直接复制念贴，不需要二次排版，而且对于Markdown很友好的一点就是Markdown的支持平台在日益增多，各平台通用的语言标记，通用的显示方式，觉得这是最大的特点。

    ​ 一直和喜欢OneNote的“颜色”和“文档分类”方式，但是自有的微软格式，真是让人头疼，完全自家的应用内还算可以，但凡要copy到第三方平台，就是面目全非，几乎都要重新排版，而对于Markdown的排版来说，转移到其他平台，如果格式不支持，起码的前后，大小都是匹配的。

    ​ 支持Markdown的平台还在日益增多，在2015年锤子科技发布会上，罗老师宣布了Markdown进入了“锤子便签”可选项，这让我觉得Markdown的支持正在越来越大众化。

    ​ 未体验过Markdown的人第一次上手肯定很麻烦，图片传输的问题，标记记不住等等的问题，但是用时间长了就会发现，它像你最终伴侣一样，看着没那么好看，但是越久越能发现他的好。

Typora 和 Markdown

简单来说，Typora 就是可以使用 Markdown 语法的编辑器。

  • Typora 使用Markdown的主要介绍希望可以直接看 Typora官网动图来的更实在

    连接在这里：https://typora.io/#feature

Typora 的使用感受

​ 轻量、颜值、实时预览、字体友好

  • Typora 的体积只有 24.5MB ，在macOS上来说已经算是轻量了
  • Typora 是真的好看，简单简约，该有的功能都有，该有的样式也不缺，简单的文字编辑工具也没什么卡顿感
  • Typora 配合 iCloud 真香，iCloud负责存储数据，可同步Apple自家设备或者任何有安装iCloud的平台
  • 这种实时预览的感觉，给写作带来了很好的视觉效果，不会在满是Markdown代码的海洋中，一点一点的捋自己的思绪
  • macOS 端的视觉感受很好，以方便得益于macOS的Retina显示屏另一方便就是Typora在macOS端的字体独到
  • typora就像win换到了macOS一样，什么都不管直接做你的工作就好，其他杂碎的事情仿佛与你无关
